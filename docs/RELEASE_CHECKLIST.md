## GitHub Actions Release

### Manual Trigger
- [ ] Push the version bump commit to `main`
- [ ] Navigate to GitHub ‚Üí Actions ‚Üí "Release" workflow
- [ ] Click "Run workflow" ‚Üí enter version (or leave empty to use package.json)
- [ ] Monitor the workflow for all checks to pass
- [ ] Workflow publishes artifacts as **draft release** (no notifications)

### Required Secrets
- `GH_TOKEN` or `GITHUB_TOKEN` with `contents: write` permission
- Set in GitHub ‚Üí Settings ‚Üí Secrets and variables ‚Üí Actions

### Optional Secrets (signing)
For signed releases (`signed=true`):

**PFX signing (traditional):**
- `CSC_LINK` ‚Äî base64-encoded PFX file content or file path
- `CSC_KEY_PASSWORD` ‚Äî PFX certificate password
- `WIN_CSC_LINK` ‚Äî Windows-specific PFX override (optional)
- `WIN_CSC_KEY_PASSWORD` ‚Äî Windows-specific PFX password (optional)

**Azure Trusted Signing:**
- `AZURE_TENANT_ID`
- `AZURE_CLIENT_ID`
- `AZURE_CLIENT_SECRET`
- `AZURE_TRUSTED_SIGNING_ACCOUNT_NAME`
- `AZURE_TRUSTED_SIGNING_CERT_PROFILE_NAME`
- `AZURE_TRUSTED_SIGNING_ENDPOINT`

**Do not commit certificate files or passwords.**

## SmartScreen & Code Signing

### Unsigned Releases
- Windows SmartScreen will warn: "Windows protected your PC"
- Users must click "More info" ‚Üí "Run anyway" to install
- This is expected for unsigned builds

### Signed Releases
- Signing with a valid OV/EV code signing certificate reduces warnings
- SmartScreen reputation still needs time and download volume
- First-time signed releases may still show warnings until enough users download

### Code Signing Options

1. **OV Code Signing** ‚Äî standard option, works immediately, costs ~$200-400/year
2. **EV Code Signing** ‚Äî higher trust, requires hardware token, costs ~$300-600/year
3. **Azure Trusted Signing** ‚Äî cloud-based, no hardware token, pay-per-signature

### Certificate Security
- Certificate files (.pfx/.p12) must **never** be committed
- Certificate passwords must **never** be stored in source
- Use GitHub Secrets or CI environment variables only
- Set `FORCE_CODE_SIGNING=true` in CI to fail if signing is unavailable
