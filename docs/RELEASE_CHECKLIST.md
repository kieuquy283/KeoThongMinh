# KeoBot Release Checklist

## Pre-Release

### Version Bump
- [ ] Update `desktop/package.json` version
- [ ] Update `frontend/package.json` version (if applicable)
- [ ] Update `backend/pyproject.toml` or `backend/__init__.py` version (if tracked)
- [ ] Run `cd desktop && npm run prepare:release` to verify version consistency
- [ ] Commit version bump and tag (`git tag v<version>`)

### Backend Validation
- [ ] `cd backend && python -m pytest` — all tests pass (including `test_distribution.py`)
- [ ] `cd backend && python scripts/validate_backend.py` — PASS
- [ ] `cd backend && python scripts/build_backend_exe.py` — build succeeds
- [ ] `cd backend && python scripts/smoke_backend_exe.py` — PASS

### Frontend Validation
- [ ] `cd frontend && npm run typecheck` — no errors
- [ ] `cd frontend && npm run build` — build succeeds

### Desktop Validation
- [ ] `node --check desktop/main.js` — syntax OK
- [ ] `node --check desktop/preload.js` — syntax OK
- [ ] `node --check desktop/localWakeWord.js` — syntax OK
- [ ] `node --check desktop/services/logger.js` — syntax OK
- [ ] `cd desktop && npm run build` — build succeeds
- [ ] `cd desktop && npm run smoke` — smoke test PASS
- [ ] `cd desktop && npm run validate:artifacts` — all checks pass
- [ ] `cd desktop && npm run check:updates` — update metadata validated

### Full Validation
- [ ] `cd desktop && npm run validate:all` — all checks pass

## GitHub Actions Release

### Manual Trigger
- [ ] Push the version bump commit to `main`
- [ ] Navigate to GitHub → Actions → "Release" workflow
- [ ] Click "Run workflow" → enter version (or leave empty to use package.json)
- [ ] Monitor the workflow for all checks to pass
- [ ] Workflow publishes artifacts as **draft release** (no notifications)

### Required Secrets
- `GH_TOKEN` or `GITHUB_TOKEN` with `contents: write` permission
- Set in GitHub → Settings → Secrets and variables → Actions

## Manual QA
- [ ] Normal text chat works
- [ ] Normal voice chat works
- [ ] Stop speaking (interrupt) works
- [ ] Hotkey interrupt works
- [ ] Wake word interrupt works
- [ ] Wake resumes after response
- [ ] Reminder while minimized works
- [ ] Tray show/hide/quit works
- [ ] Start with Windows toggle works
- [ ] Mascot state changes track app state
- [ ] Tools/provider diagnostics show correct status
- [ ] App restart preserves settings

## Asset & Secret Check
- [ ] All mascot assets are in `frontend/dist/keobot/` after build
- [ ] No API keys in source code or config files
- [ ] No `.env` files included in package
- [ ] `node_modules` not bundled in release package (only native modules)

## Update Config Check
- [ ] `electron-updater` publish config set to GitHub Releases provider
- [ ] `build.publish` in `desktop/package.json` has correct owner/repo
- [ ] `releaseType: "draft"` — no automatic notifications
- [ ] No tokens in publish config (secrets come from CI env)
- [ ] `latest.yml` generated by electron-builder at build time
- [ ] Update status shows "Ready to check updates" when publish configured
- [ ] Update status shows "Auto update is not configured for this build" when publish is null (dev builds)
- [ ] Dev builds do not attempt to check for updates

## Windows Startup Check
- [ ] Start with Windows setting works
- [ ] Background assistant setting works

## Wake Word Check
- [ ] Web Speech API wake word works
- [ ] Local Porcupine wake word works (if Picovoice key configured)
- [ ] Hotkey fallback works

## Rollback Plan
- [ ] Previous release artifacts are backed up
- [ ] Configuration is backward-compatible
- [ ] Downgrade path tested (install older version over newer)
- [ ] Draft release can be deleted without affecting users

## Post-Release
- [ ] Promote draft → published (remove "draft" flag) after QA passes
- [ ] Tag release in git (already done in version bump step)
- [ ] Update release notes on GitHub
- [ ] Verify update metadata is reachable from the release

## SmartScreen Warning
The first few releases will trigger Windows SmartScreen since the executable is unsigned. Users must click "More info" → "Run anyway" to install. Consider code signing for future releases.
